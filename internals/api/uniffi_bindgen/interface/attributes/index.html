Attribute definitions for a ComponentInterface.

This module provides some conveniences for working with attribute definitions from WebIDL. When encountering a weedle ExtendedAttribute node, use TryFrom to convert it into an Attribute representing one of the attributes that we support. You can also use the parse_attributes function to parse an ExtendedAttributeList into a vec of same.

We only support a small number of attributes, so it’s manageable to have them all handled by a single abstraction. This might need to be refactored in future if we grow significantly more complicated attribute handling.
